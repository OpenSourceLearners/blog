module.exports和exports的区别

发表于 2019-02-18 | 分类于 web前端

问题

module.exports和exports都是定义模块的方法。
但是两个一起使用的时候，如果使用了module.exports时exports就会失效。

原因

因为exports是引用module.exports，而且module.exports是{}(空对象)。
如果将exports重新赋值的时候就与module.exports就断了联系。
最总模块导出的是module.exports，最后导出的是空对象。
